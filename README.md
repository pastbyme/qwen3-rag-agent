# Qwen3-Agentic-RAG: 基于 Qwen3 的智能体检索增强生成系统

本系统在传统 RAG（检索增强生成）的基础上引入了智能体（Agent）决策层，实现了从“静态检索生成”到“动态思考、工具调用与自我纠错”的范式升级。本说明文档将详细解析该实验的核心实现机制与执行流程。

---

## ⚙️ 核心实现流程

系统的完整执行链路分为五个关键阶段，由 `AgentCore` 作为中央引擎进行统一编排，形成闭环流水线：

### 1. 意图识别与任务分类 (Task Classification)
当系统接收到用户的 `Query` 时，并不是直接去检索数据库，而是首先交由 `TaskClassifier` 处理。
- **实现机制**：利用 Qwen3 大模型结合强约束的 System Prompt，对问题进行初步的语义和逻辑分析。
- **输出结果**：模型输出一个结构化的 JSON 对象，包含：
  - **任务类型**（如 `simple_qa` 简单问答, `calculation` 数学计算, `multi_hop` 多跳推理）。
  - **任务复杂度预测**。
  - **所需外部工具列表**（`required_tools`）。
  - **建议的检索参数**（`suggested_top_k` 和相似度阈值）。

### 2. 动态知识检索 (Dynamic Retrieval)
不同于传统 RAG 固定的检索参数，本系统实现了自适应检索。
- **实现机制**：`Retriever` 模块接收第一步任务分类器传入的动态参数进行执行。
- **执行逻辑**：对于需要跨文档关联的复杂推理任务，系统会自动调大 `top_k` 以获取更广泛的上下文；对于单一事实问答，则减小 `top_k` 以降低噪声干扰。初筛完成后，`Reranker` 会根据关键词密度等规则对文档片段进行二次重排。

### 3. 确定性工具调度 (Tool Dispatch)
为了解决大模型在绝对精确任务（如数学计算）上的“幻觉”缺陷，系统引入了外部工具链。
- **实现机制**：`ToolRegistry`（工具注册中心）以插件化形式封装了多个外部执行函数（如 `calculate` 计算器、`summarize` 摘要器）。
- **执行逻辑**：若第一步解析出当前任务需要特定工具，Agent 将提取关键参数交由对应的 Python 函数直接执行。工具返回的绝对正确结果将作为强事实被注入上下文中，替代大模型的概率预测。

### 4. 答案生成与置信度门控重试 (Generation & Retry Mechanism)
本环节是 Agent 区别于传统 RAG 的核心“自我反思与纠错”中枢。
- **实现机制**：`Generator` 整合（1）原始问题、（2）动态检索文档、（3）工具执行结果，指导 Qwen3 生成最终答案。同时，强制要求模型输出对该答案的**置信度评分（Confidence，0.0~1.0）**。
- **执行逻辑**：系统预设了严格的置信度门控阈值（如 `0.7`）。
  - 若 `Confidence >= 0.7`：认为答案可靠，直接输出。
  - 若 `Confidence < 0.7`：自动触发 `RetryManager`，系统会自动调整 Prompt（例如过滤掉相似度偏低的上下文，或要求模型换种思路），并重新生成答案。重试次数设有上限（默认 3 次），以此兜底系统的输出质量。

### 5. 长短期记忆管理 (Memory Management)
- **实现机制**：在确认生成最终答案后，将其交由 `MemoryManager` 维护跨会话状态。
- **执行逻辑**：系统基于“时间衰减（Recency）”与“访问频率（Frequency）”联合计算记忆重要度。高价值的交互会被持久化为历史上下文，在下一次问题分类时被动态载入；达到容量上限时，采用 LRU（最近最少使用）策略自动淘汰低频记忆。

## 📂 核心代码目录映射

上述五个流程分别对应以下模块化的代码结构：

```text
├── agent/
│   ├── agent_core.py          # 中央调度引擎，串联控制上述 5 个流程
│   ├── task_classifier.py     # [流程1] 意图识别与动态参数推导
│   ├── tool_registry.py       # [流程3] 外部工具注册与安全执行
│   └── memory.py              # [流程5] 全局长短期记忆状态机维护
├── rag/
│   ├── retriever.py           # [流程2] 基于 BGE 模型的动态向量检索
│   ├── reranker.py            # [流程2] 文档二次重排序
│   └── generator.py           # [流程4] 带置信度门控的生成与重试管理器
├── models/
│   └── qwen3_client.py        # 大模型底层通信封装（对接本地 Ollama API）
├── metrics.py                 # 自动化指标评测脚本
└── config.py                  # 系统全局参数配置
```

## 🚀 运行与测试

在安装好相关的 Python 依赖（`openai`, `sentence-transformers`, `numpy` 等）并启动本地 Qwen3 服务后，可以直接运行评估脚本。系统将自动化遍历包含各类任务的测试集，并打印详细的执行流转步骤与最终指标。

```bash
python metrics.py
```
